# MVP

## MVP Goal

Prove that AI Delivery Engine can coordinate a small set of AI-assisted roles around a shared backlog without building a large platform.

## In Scope

- documentation for the operating model
- defined roles for PO/PM, UX/UI, Front-end, Back-end, QA, and optional Tech Lead
- a backlog-first workflow
- manual review and approval rules
- a recommended path toward a first local implementation

## Out Of Scope

- real agent execution
- external API calls
- GitHub automation
- database design
- production UI

## What Is Manual In V1

- writing prompts
- running AI tools
- reviewing outputs
- editing backlog items
- deciding readiness

## What Can Be Automated Later

- prompt execution
- schema validation
- backlog item creation
- issue synchronization
- QA checklist generation

## MVP Success Criteria

- a new contributor can understand the system by reading the docs
- the team can describe one end-to-end flow from brief to approved backlog
- V1 boundaries are explicit enough to prevent premature implementation
