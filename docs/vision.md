# Vision produit

AI Delivery Engine a pour ambition de devenir un moteur de livraison logiciel assisté par IA.

Le projet doit permettre de passer progressivement de :

```txt
Une idée exprimée en langage naturel
```

à :

```txt
Un backlog structuré, priorisé, challengé et prêt à être développé
```

puis, dans les versions suivantes :

```txt
Des issues GitHub, des prompts Codex, des pull requests et des validations QA
```

## Pourquoi ce projet existe

Les outils IA savent déjà générer du code, écrire des spécifications ou proposer des tests. Le problème principal est l'orchestration :

- qui décide quoi faire ?
- dans quel ordre ?
- avec quel contexte ?
- comment éviter de refaire le même travail à chaque projet ?
- comment garder une trace exploitable dans GitHub ?

AI Delivery Engine répond à ce besoin en structurant les interactions entre agents IA autour d'un backlog et d'un dépôt GitHub.

## Positionnement

Le projet n'est pas :

- un simple chatbot ;
- un simple workflow n8n ;
- un générateur de code automatique sans contrôle ;
- un outil magique censé remplacer tout le delivery.

Le projet est :

- un orchestrateur d'agents spécialisés ;
- un moteur de backlog ;
- un assistant de cadrage, développement et QA ;
- un cockpit de delivery augmenté par IA.

## Cas d'usage cible

Le premier cas d'usage est la création d'un site vitrine freelance web / IA.

Ce cas permet de tester toute la chaîne :

1. expression du besoin ;
2. clarification produit ;
3. génération du backlog ;
4. découpage en tâches techniques ;
5. création d'issues GitHub ;
6. préparation des prompts Codex ;
7. suivi de la livraison.

## Ambition long terme

À terme, AI Delivery Engine doit pouvoir être utilisé pour :

- lancer rapidement un nouveau projet client ;
- auditer un projet existant ;
- maintenir un backlog vivant ;
- challenger les besoins entrants ;
- créer des issues GitHub propres ;
- préparer des prompts d'implémentation ;
- assister la QA et la revue de code ;
- capitaliser sur les projets précédents.
